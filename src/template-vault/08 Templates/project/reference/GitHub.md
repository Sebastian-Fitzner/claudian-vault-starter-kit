## Worktree Convention

- Use a dedicated `git worktree` for task branches when the task sets `worktree: true`
- Create worktrees as sibling folders of the main checkout
- Folder pattern: `<repo-name>-<branch-slug>`
- `branch-slug` = branch name lowercased with `/` replaced by `-`
